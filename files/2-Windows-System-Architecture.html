2-Windows System Architecture



Environment Subsystems

Kullanıcı uygulamaları direkt olarak Windows Sistem Servislerine ulaşamaz ve bunun yerine bir veya birden fazla subsystem dll kullanır. Mesela windows uygulaması SUA(POSIX,psxdll.dll) subsystem'i kullanamaz. Windows uygulaması Windows subsystem DLLs kullanır.(Kernel32.dll,advapi32.dll,User32.dll,Gdi32.dll).

Session Manager

Subsystems(altsistemler) , Session Manager Subsystems (Smss.exe) proses'i tarafından başlatılır.Subsystem startup bilgileri "HKLM\SYSTEM\CurrentControlSet\Control\SessionManager\SubSystems" altında saklanır. Burada Windows ve Debug string değerleri bulunuyor. Windows , değer olarak subsystem proses'i olan (Client/Server Runtime Subsystem)Csrss.exe'i alır. Proses/Thread yönetimi yapan , WinAPI dll'leri import eden ve sistem kapanmasında GUI'ın kapatılmasını sağlar. Debug değeri ise boş çünkü WinXP'den beri gerek duyulmuyor. Optional kısmı ise SUA'nın Win10 da olmamasından dolayı boş.Fakat olsaydı Psxss.exe yazıyor olurdu yani POSIX subsystem proses.

Wininit.exe - Windows Initialization Process

smss.exe tarafından oluşturuluyor. "Username : NT Authority\System" , user-mode tarafında görevlerin başlatılmasını sağlar. Wininit.exe Parent process olup child process olarak altında services.exe(service control manager ya da Scm) bulunur. Lsass.exe(Local Security Authentication Subsystem Service)'ı başlatır.

Services.exe - Service Control Manager

Parent prosesi wininit.exe'dir. Services.exe "svchost.exe, dllhost.exe, taskhost.exe spoolsv.exe" gibi proseslerin parent prosesidir. Servisler kayıt defterinde "SYSTEMCurrentControlSetServices" içinde tanımlanmıştır. Servisleri hafızaya yükler. Sadece bir tane services.exe prosesi çalışmalıdır.

Lsass.exe - Local Security Authority

Wininit.exe'nin child prosesidir. Child prosesi olmamalıdır. Sadece bir tane lsass.exe prosesi çalışmaktadır. "Local security policy" den sorumludur bunun içinde kullanıcıların yönetimi "login,password" policies den sorumludur ve bunları security event log kayıtlarına yazar.

svchost.exe - Service Hosting Process

Bir'den fazla svchost.exe çalışıyor olabilir. "Username" olarak "NT Authority\System,Local Service,Network Service" olmalıdır. Parent proesesi "services.exe". Komut satırında "svchost -k " olarak çalışır. svchost.exe birden çok Windows servisini çalıştıran bir prosestir.

WINLOGON.exe - Windows Logon Process

Parent prosesi yoktur. "Username NT Authority\System". Child prosesi olabilir. Kullanıcı parolası girildikten sonra doğrulama içi LSASS(lsass.exe)'e gönderilir ve Active Directory veya SAM üzerinden doğrulama yapılır.

explorer.exe - Windows Explorer

Parent prosesi yok. "Username : 'Giriş yapılan kullanıcı' ". Child prosesleri olur.Windows gezgini olarak da adlandırılabilir, açılan pencereler child proses olarak altında gözükebilir.
